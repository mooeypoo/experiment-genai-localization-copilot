About This Experiment

This experiment examines how an AI coding assistant (GitHub Copilot) makes decisions when incrementally adding internationalization and localization features to a Vue.js application. By documenting each step's decisions, tradeoffs, and implementation choices, we can observe what the AI prioritizes, what it overlooks, and how it adapts to evolving constraints.

Purpose

Traditional software localization involves many subtle decisionsâ€”message structure, pluralization rules, text direction, cultural adaptation, and more. By having an AI assistant build this incrementally, we can:

  • Observe which localization patterns the AI chooses naturally
  • Identify what the AI misses or implements incorrectly
  • Understand how explicit prompts shape implementation quality
  • Document real-world decisions and tradeoffs in i18n development

How It Works

Each step in this experiment is a git tag (step-01, step-02, etc.) representing a specific milestone in the app's development. Every step includes:

  • Agent Notes: Detailed documentation of decisions, rationale, ambiguities, tradeoffs, and known issues
  • Prompt: The instruction given to the AI assistant (when available)
  • Working Code: A fully functional snapshot of the app at that stage

Navigation Tip: When viewing a step, the sidebar shows the agent notes and prompt for that specific step. Use the step selector in the top bar to navigate between different stages of development.

The Application

The demo app is a simple social feed built with Vue 3, starting as a basic English-only prototype and evolving into a fully internationalized application supporting multiple languages, including right-to-left scripts, locale-aware formatting, and cultural considerations.

Technical Implementation

This GitHub Pages site is built from git tags using a custom build pipeline. Each step's code remains frozen in history, while the global shell (navigation, sidebar, styling) can be updated independently. This ensures historical steps remain accurate exhibits while allowing the viewing experience to improve over time.
